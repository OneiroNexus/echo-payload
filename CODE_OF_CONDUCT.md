# Code of Conduct

echo-payload is a technical research resource. Contributors are expected to:

1. **Be precise** — cite sources for benchmark numbers, use correct terminology
2. **Be constructive** — suggest improvements, not just criticisms
3. **Be inclusive** — this repo serves researchers across all backgrounds and institutions
4. **Be honest** — don't submit fabricated benchmark results

Issues and PRs that violate these principles will be closed without comment.
